# اقرأ هذا أولاً — قبل الرفع لأي منصة استضافة

## ليه ظهرت "There is nothing here yet"؟

هذا النظام **مش موقع ثابت (static website)**. هو تطبيق متكامل (Full-Stack) مكوّن من 3 أجزاء لازم تشتغل مع بعض:

| الجزء | الوظيفة | لازم يشتغل بشكل مستقل |
|---|---|---|
| **PostgreSQL** | قاعدة البيانات (كل الجداول، الـ BOQ، الموافقات، الـ audit log) | نعم — سيرفر قاعدة بيانات |
| **Backend (Node.js)** | الـ API اللي يطبّق كل قواعد العمل (البوابة الموافقات، تتبع المشتريات...) | نعم — سيرفر Node شغال باستمرار |
| **Frontend (React)** | الواجهة اللي تشوفها في المتصفح | يحتاج build، وبعدين لازم يتكلم مع الـ Backend |

لو رفعت المجلد كامل (أو حتى مجلد `frontend` لوحده) لمنصة استضافة ثابتة زي Netlify/Vercel بدون إعداد Backend + قاعدة بيانات، هتشوف صفحة فاضية أو "nothing here" — لأن:
1. ملفات المصدر (`src/main.jsx`) لازم تتبني (build) بـ Vite الأول، مش تفتح مباشرة كـ HTML.
2. حتى لو اتبنت، الواجهة هتحاول تتصل بـ `/api/...` ومفيش Backend شغال ترد عليها.

## الحل — تم تجهيزه في هذا الملف

1. ✅ **`frontend/dist/`** — نسخة الإنتاج المبنية جاهزة (HTML/CSS/JS ثابتة). لو رفعتها لوحدها لاستضافة ثابتة هتظهر واجهة تسجيل الدخول فعليًا (مش صفحة فاضية) — لكن العمليات (تسجيل دخول، بيانات) مش هتشتغل من غير Backend + قاعدة بيانات.
2. ✅ **تم اختبار النظام بالكامل فعليًا** — قاعدة البيانات، تسجيل الدخول، دورة حياة الـ BOQ كاملة (إنشاء → مراجعة → موافقة الإدارة → إرسال للعميل)، المشتريات (PR → عرض أسعار → PO)، المخزن (استلام → فحص → إصدار)، التنفيذ، المالية، ولوحات التحكم الثلاثة — كل حاجة اشتغلت بنجاح على السيرفر الحقيقي.

## طريقة التشغيل الصحيحة (محليًا أو على سيرفر)

```bash
# 1. شغّل قاعدة البيانات
docker compose up -d

# 2. حمّل الهيكل والبيانات التجريبية
psql postgresql://erp_user:erp_pass@localhost:5432/erp_db -f backend/db/schema.sql
psql postgresql://erp_user:erp_pass@localhost:5432/erp_db -f backend/db/seed.sql

# 3. شغّل الـ Backend
cd backend && cp .env.example .env && npm install && npm run dev

# 4. شغّل الـ Frontend (تطوير) أو استخدم frontend/dist (إنتاج)
cd frontend && npm install && npm run dev
```

## لو عايز ترفعه لاستضافة حقيقية (Production)

تحتاج 3 خدمات منفصلة على المنصة اللي هتستخدمها (مثل Railway, Render, DigitalOcean App Platform):
- **Database service**: PostgreSQL managed instance
- **Backend service**: يشغّل `node src/server.js` مع `DATABASE_URL` صح
- **Frontend**: استضافة `frontend/dist` (ثابت) مع إعادة توجيه `/api` لعنوان الـ Backend

منصات زي Netlify/Vercel لوحدها تناسب الـ Frontend بس — مش هتشغّل الـ Backend والداتابيز، فلازم Backend + DB على خدمة منفصلة (Render/Railway وغيرها كويسة وسهلة للبداية).

تفاصيل أكتر في `README.md`.
